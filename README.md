# Лабораторно-практичне заняття №1
## Тема: Програмні моделі систем координат

## 1. Короткий опис роботи
**Мета:** Спроектувати та реалізувати імутабельні програмні моделі для представлення точок у 2D та 3D системах координат, а також провести аналіз продуктивності обчислень.

**Реалізовані класи:**
* **Models:** `CartesianPoint2D`, `PolarPoint`, `CartesianPoint3D`, `SphericalPoint` — імутабельні структури (`readonly struct`), що забезпечують цілісність даних.
* **Services:** `DistanceCalculator` — містить логіку обчислення відстаней (Евклідова, за теоремою косинусів, хордова та дугова).
* **Benchmarking:** `BenchmarkRunner` — інструмент для заміру швидкодії на масивах по 100,000 пар точок.

---

## 2. Інструкції для запуску проекту
Для роботи проекту необхідна встановлена платформа **.NET 10** (або новіша).

1.  **Клонування репозиторію:**
    ```bash
    git clone [https://github.com/volandr1/CoordinateSystemsLab.git](https://github.com/volandr1/CoordinateSystemsLab.git)
    ```
2.  **Перехід у папку проекту:**
    ```bash
    cd CoordinateSystemApp
    ```
3.  **Запуск через термінал:**
    ```bash
    dotnet run
    ```
Або відкрийте файл `CoordinateSystemApp.sln` у **JetBrains Rider** та натисніть `Shift + F10`.

### Аналіз та висновки:
* **Для 2D:** Обчислення в декартовій системі значно швидші. Використання теореми косинусів у полярних координатах потребує виклику `Math.Cos`, що створює додаткове навантаження на процесор порівняно з простою арифметикою.
* **Для 3D:** Найшвидшим методом є **3D Cartesian**. Найповільнішим — **3D Spherical (Arc)**, оскільки формула великої колової відстані (Great Circle Distance) є найбільш математично складною та містить багато тригонометричних функцій.
