# Лабораторно-практичне заняття №1
**Тема:** Програмні моделі систем координат

## 1. Короткий опис роботи
**Мета:** Спроектувати та реалізувати імутабельні програмні моделі для представлення точок у 2D та 3D системах координат, а також провести порівняльний аналіз продуктивності обчислень.

**Реалізовані класи:**
* **Models:** `CartesianPoint2D`, `PolarPoint`, `CartesianPoint3D`, `SphericalPoint` — незмінні структури (`readonly struct`), що забезпечують цілісність даних.
* **Services:** `DistanceCalculator` — клас із логікою розрахунку відстаней (Евклідова, за теоремою косинусів, хордова та дугова).
* **Benchmarking:** `BenchmarkRunner` — модуль для навантажувального тестування на 10,000,000 ітерацій.

---

## 2. Інструкції для запуску проекту
Для роботи проекту необхідна встановлена платформа **.NET 10** (або новіша).

1.  **Клонування репозиторію:**
    ```bash
    git clone [https://github.com/volandr1/CoordinateSystemsLab.git](https://github.com/volandr1/CoordinateSystemsLab.git)
    ```
2.  **Перехід у папку проекту:**
    ```bash
    cd CoordinateSystemApp
    ```
3.  **Запуск через термінал:**
    ```bash
    dotnet run
    ```
Проект також можна відкрити та запустити через **JetBrains Rider**.

## 3. Результати перевірки коректності
Програма виконує тестові перетворення (конвертація туди-назад). Результати тестування підтверджують правильність реалізації формул.

**Дані з консолі:**
* **2D Recovered:** Cartesian2D(X=10,00, Y=10,00) — **Помилка: 0,000000**
* **3D Recovered:** Cartesian3D(X=6,12, Y=6,12, Z=5,00) — відновлено успішно.

## 4. Результати аналізу продуктивності (Бенчмаркінг)
Тестування проведено на масиві з **10,000,000** ітерацій.

| Метод обчислення | Час виконання (мс) |
| :--- | :--- |
| **2D Декартова відстань** | 272 мс |
| **2D Полярна відстань (Теорема косинусів)** | 184 мс |
| **3D Декартова відстань** | 113 мс |
| **3D Сферична відстань** | 63 мс |

### Аналіз та висновки:
* **Для 2D:** На диво, обчислення в полярних координатах (184 мс) виявилися швидшими за декартові (272 мс). Це свідчить про надзвичайно ефективну оптимізацію тригонометричних функцій у середовищі .NET 10 на даній архітектурі процесора.
* **Для 3D:** Сферична відстань (63 мс) продемонструвала майже двократну перевагу над декартовою (113 мс). Це підтверджує, що сучасні JIT-компілятори здатні ефективно векторизувати складні математичні операції.

## 5. Загальний висновок
В ході лабораторної роботи було закріплено навички створення **імутабельних структур** на C#. Ключовим інсайтом став той факт, що теоретично "важкі" математичні функції (sin, cos) у сучасних високорівневих мовах програмування можуть працювати швидше за множинні операції з плаваючою комою у декартовому просторі. Вибір системи координат має базуватися на реальних тестах продуктивності під конкретну платформу.
